# Proposal: Clarify Note Status
**Status: Ready for Review**

## Intent
The current description of `Status: Note` implies it still requires structure (an `Intent` section and an `Unresolved` section). In practice, a Note should be a freeform space to capture thoughts and discussion before a formal proposal takes shape — no mandatory headings, no template to fill in.

## Scope
- **In scope**: Wording around the `Note` status in the Propose section of PROCESS.md; the proposal template's status line
- **Out of scope**: Any changes to Draft, Ready for Review, or Approved statuses

## Specification Deltas

### MODIFIED
- The `Note` description changes from "a brief `Intent` and an `Unresolved` section are all that is required" to making clear that no section headings are required — a Note is freeform prose (or bullet points) capturing early thoughts and discussion
- The proposal template should reflect that a Note need not follow the standard heading structure
