# Process Task Discipline

## Intent
Two gaps in the current process guidance:

1. Tasks must be written to the change file before approval to proceed is sought — currently the process doesn't make this explicit.
2. Tasks must be ticked off one by one as they are completed, not all at once at the end — agents sometimes batch-complete tasks retrospectively.

## Plan
- [x] CHANGE: `AGENT/PROCESS.md` — clarify that the Plan must be written to the change file before approval is sought, and that tasks are ticked off one by one as completed

## Conclusion
Updated Plan guidance in `PROCESS.md` to make both requirements explicit.
