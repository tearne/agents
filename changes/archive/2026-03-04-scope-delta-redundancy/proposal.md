# Proposal: Scope/Delta Redundancy
**Status: Ready for Review**

## Intent
The `Scope` and `Specification Deltas / ADDED` sections overlap for additive changes — both end up listing the same new things. Reordering and reframing eliminates the redundancy while keeping both sections useful.

## Scope
- **In scope**: the proposal template in PROCESS.md
- **Out of scope**: the design template; existing archived proposals

## Specification Deltas

### MODIFIED
- `Specification Deltas` moves above `Scope` in the proposal template, so precise changes are stated before the boundary discussion
- `Scope` is reframed to focus on the out-of-scope boundary; the "In scope" list becomes optional (omit when the Delta already makes it clear)
